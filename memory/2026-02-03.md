**The Big Discovery:** All background processes launched by AI agents via `exec &` die silently after ~20-30 minutes. Parent session cleanup sends SIGTERM to children. Signal handlers calling `sys.exit(0)` make crashes look like clean exits. This affects every Clawdbot/OpenClaw/Claude Code user.

**Root Causes Found (BTC bot crashed 3x):**
1. `asyncio.gather(*tasks)` without `return_exceptions=True` ‚Äî copied from v3 bug
2. Exec session lifecycle kill ‚Äî processes inherit parent session, die when session cleaned up
3. No autonomous monitoring ‚Äî nobody knows until manually asked

**Solution Built ‚Äî 3 Layers:**
1. Detached launch: `setsid` + `nohup` + `disown` (PPID=1, own SID)
2. `managed-process.sh` framework: register ‚Üí start ‚Üí status ‚Üí healthcheck
3. Cron every 5min: auto-detect dead processes, restart, alert
