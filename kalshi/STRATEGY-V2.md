# Kalshi 策略 V2 — 从 Putin 教训中总结

## ⚠️ 核心工作原则：先搜索互联网再动手
任何新功能、新策略、新工具 → 先搜索现有方案、已有工具、别人的做法，再写代码。

## 核心问题

Putin YES @ 25¢，事件已经发生，合约规则明确覆盖，但我们完全没发现。
这不是扫描列表不全的问题，是**整个策略方向的问题**。

## V1 策略的问题

V1 思路：扫描市场价格 → 找极端值 → 研究验证
- ❌ 只看价格面，不看事实面
- ❌ 从市场出发找机会，但市场有几千个，大海捞针
- ❌ Junk bond 回报太低（5-15%），真正的大机会（4x）被忽略
- ❌ 跨平台对比失败：Kalshi 和 Polymarket 市场定义不同，无法自动匹配

## 真正的 Alpha 来源

回顾 Jason 的成功案例，alpha 来自三个地方：

### 1. 信息速度差 (Information Lag)
- Trump-Putin 通话 → 主流媒体已报道 → Kalshi 价格没反映
- 原因：Kalshi 散户不看新闻，或看了不知道这个市场存在
- **策略：新闻先行，市场后找**
  - 监控突发新闻 → 匹配到受影响的 Kalshi 市场 → 快速交易
  - 翻转逻辑：不是"看市场找新闻"，而是"看新闻找市场"

### 2. 规则理解差 (Rules Edge)
- "Meeting" 的定义包括 phone call — 大多数交易者没读 PDF
- SCOTUS 休庭期 — 散户不知道法院日程
- CR 后备方案 — 散户不了解国会流程
- **策略：成为规则专家**
  - 下载并分析所有活跃市场的合约 PDF
  - 建立"规则数据库"：每个市场的关键定义和边界条件
  - 当事件发生时，快速判断是否触发结算

### 3. 时间窗口差 (Expired Window)
- "January" 市场在 February 1 日应该可以确定结果
- 但市场还在交易，价格没调整
- **策略：到期窗口扫描**
  - 找时间窗口已过的市场（1月的市场，现在2月了）
  - 验证事件是否在窗口内发生
  - 如果已发生但价格没反映 → 套利

## V2 策略设计

### 策略 A：新闻驱动（News → Market）
```
流程：
1. 每小时扫描 Google News 头条（政治、经济、外交类）
2. 提取关键实体和事件（谁做了什么）
3. 匹配到 Kalshi 活跃市场
4. 检查合约规则确认是否触发
5. 如果市场价格没反映 → 通知 Jason
```
优势：直接找到"事实已发生但市场没反映"的情况
难点：新闻到市场的匹配需要比关键词更好的方法

### 策略 B：到期窗口套利（Expired Window Arbitrage）
```
流程：
1. 找所有时间窗口刚过的市场（标题含"January"且已到2月）
2. 对每个市场，搜索事件是否在窗口内发生
3. 查合约 PDF 确认定义
4. 价格 vs 事实不符 → 通知 Jason
```
优势：最容易验证（事实已经确定），风险最低
难点：机会窗口短（结算前几天）

### 策略 C：规则套利（Rules Arbitrage）
```
流程：
1. 下载所有活跃市场的合约 PDF
2. 提取关键定义和边界条件
3. 当相关新闻出现时，快速判断是否满足规则
4. 重点关注定义模糊或反直觉的规则（如 phone call = meeting）
```
优势：长期竞争优势，别人不会做这个功夫
难点：需要大量前期工作解读 PDF

### 策略 D：事件日历预判（Event Calendar）
```
流程：
1. 维护一个事件日历（数据发布、法院日程、国会日程、外交安排）
2. 事件前 1-3 天提前关注相关市场
3. 事件发生后第一时间交易
```
优势：有准备地等待，不是盲目扫描
难点：需要手动维护日历

## 建议优先级

1. **策略 B（到期窗口）** — 最容易实现，最低风险，Putin 就是这种机会
2. **策略 A（新闻驱动）** — 最大 alpha 来源，但实现复杂
3. **策略 C（规则套利）** — 长期优势，但需要前期投入
4. **策略 D（事件日历）** — 辅助工具，配合其他策略使用

## 执行计划

### 第一步：到期窗口扫描器
- 扫描所有标题含时间词的市场（January, February, Q1, before X date）
- 检查当前日期是否已过窗口
- 自动搜新闻验证事件
- 这个 Putin 机会就是这样发现的

### 第二步：新闻监控
- 每小时拉 Google News RSS（政治+经济+外交）
- 提取关键事件
- 模糊匹配到 Kalshi 市场
- 重点：不需要完美匹配，只需要发现"可能相关" → 然后人工确认

### 第三步：合约规则库
- 下载活跃政治市场的 PDF
- 提取关键定义
- 建立规则摘要数据库

## 关键认知

**从"扫描市场找机会"转向"追踪事实找错误定价"。**

市场有几千个，大部分定价合理。
但现实世界的事实只有一个版本。
当事实和价格不符时，就是我们的机会。
