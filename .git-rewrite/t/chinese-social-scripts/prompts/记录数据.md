# AI能力：记录数据

## 触发词
当Jason说"记录数据"或发布内容后报数据时触发

## 工作流程
1. **获取信息**
   - 发布日期
   - 标题
   - 选题类型（产品测评/教程/种草等）
   - 数据：播放量、点赞、评论、分享

2. **记录到表格**
   - 追加到 `04-数据/内容数据.md`
   - 格式：表格行

3. **简单分析**
   - 如果播放量明显高于平均 → 标注🔥
   - 如果数据积累到一定量（10条+），主动建议复盘

4. **移动文稿**
   - 把文稿从 `01-选题/待做/` 移到 `01-选题/已发布/`
   - 在文稿末尾附上数据

## 示例对话
```
Jason: "记录数据：冬季护肤保湿，播放3.2万，点赞540，评论38"

AI:
1. 记录到内容数据.md...
2. 移动文稿到已发布目录...
3. 回复：
   "✅ 数据已记录
   - 播放：32,000 🔥（高于平均）
   - 点赞率：1.69%
   - 评论率：0.12%
   
   这个选题表现不错！标题「干皮冬天真的会死😭这瓶救命」痛点型hook效果好。"
```

## 注意事项
- 如果数据积累到10-20条，主动建议更新方法论
- 观察规律：什么选题/标题容易爆
- 数据是方法论的基础，记录要准确
