# 小红书内容创作系统（轻量版）

> 从碎片化创作 → 系统化创作
> 参考 @dontbesilent12 的思路，专注小红书，轻量快速

---

## 核心闭环
**记录选题 → 生成文稿 → 生成标题 → 发布 → 记录数据 → 反哺方法论**

---

## 目录结构

```
01-选题/
├── 想法记录.md          # 碎片想法收集箱
├── 待做/                # 决定要做的选题
└── 已发布/              # 发布后归档（含数据）

02-素材/
├── 金句库.md            # 高质量表达、hook句式
└── 产品库.md            # 产品信息、成分、卖点

03-方法论/
├── 标题规律.md          # 从数据总结标题规律
├── 选题规律.md          # 什么选题容易爆
└── 风格指南.md          # Jason的人设和语言风格

04-数据/
└── 内容数据.md          # 每条内容的数据记录

prompts/                 # AI快捷指令定义
├── 记录选题.md
├── 生成文稿.md
├── 生成标题.md
└── 记录数据.md
```

---

## 如何使用

### 1. 记录选题
有选题想法时，直接说：
```
"记录选题：[选题方向]"
```
AI会自动记录到 `01-选题/想法记录.md`

### 2. 生成文稿
决定做某个选题时，说：
```
"写文稿：[选题名]"
```
AI会：
- 先检索素材库（金句、产品、已发布内容）
- 建议复用相关内容
- 基于你的风格生成文稿
- 保存到 `01-选题/待做/`

### 3. 生成标题
文稿完成后，说：
```
"生成标题"
```
AI会生成3个标题选项（痛点型/利益型/好奇型）+ 逻辑说明

### 4. 记录数据
内容发布后，报数据：
```
"记录数据：[选题名]，播放X万，点赞X，评论X"
```
AI会：
- 记录到 `04-数据/内容数据.md`
- 移动文稿到 `01-选题/已发布/`
- 简单分析数据表现

## 系统优势

### vs 碎片化创作
- ❌ 每次从零开始
- ❌ 不知道以前写过什么
- ❌ 好的表达散落各处

### ✅ 系统化创作
- ✅ 素材复用，不重复造轮子
- ✅ 方法论沉淀，越用越聪明
- ✅ 数据驱动，不凭感觉

## 关键原则
1. **先检索再创作** — 避免重复造轮子
2. **数据驱动** — 方法论从数据来，不瞎猜
3. **轻量快速** — 能用为先，别搞空架子
4. **持续迭代** — 每次创作都在给系统添砖加瓦
